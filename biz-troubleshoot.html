POS isn't syncing transactions to LYMX dashboard

POS
Symptom: You ran 8 LYMX transactions on Saturday, dashboard shows 0.
Fix in 5 minutes
  1. Check POS internet connection — most syncs need live network
  2. Dashboard → Integrations → check status badge for your POS (Square/Toast/Clover/Lightspeed). If "disconnected," click reconnect.
  3. Manually trigger sync: Dashboard → Settings → Force re-sync. Pulls last 7 days from POS.
  4. If still missing: contact your success coach. We can pull transactions from POS API directly.
Customers complaining about missing LYMX? Issue retroactive credits via Dashboard → Customer claims. They show up in customer wallets immediately.

"This QR code is expired" error at register

POS
Symptom: Cashier scans customer QR, gets "expired" error.
Likely causes
  1. Customer's QR refreshes every 60 seconds. Have them tap their phone or refresh wallet.
  2. Your POS system clock is off — POS Settings → Time → set to network time
  3. Internet was offline when QR was generated — retry once back online

POS asking for activation code that expired

POS
Symptom: During first-time setup, your activation code says "expired."
Fix steps
  1. Activation codes expire after 14 days for security
  2. Dashboard → Integrations → tap your POS → "Generate new activation code"
  3. Enter new code in POS within 14 days

Receipts are missing the LYMX line item

POS
Symptom: Customer paid, earned LYMX, but receipt doesn't show "+25 LYMX earned."
Fix steps
  1. POS Settings → Receipts → enable "LYMX line item"
  2. If using thermal printer that ran out of paper, change paper — LYMX line is on the bottom
  3. If receipt template was customized: re-apply LYMX template (Integrations → POS → Receipt template → Reset)

Dashboard shows wrong rate (e.g., 5/$1 when you set 3/$1)

Dashboard
Symptom: Settings show one rate, customers earning at a different rate.
Fix steps
  1. Your rate is locked at activation by the platform. To change: Dashboard → Settings → Rate change request
  2. Rate changes take effect 7 days after request (cooling period)
  3. If activation rate was wrong from day one: contact success coach to rectify within 30 days at no penalty
Important: Rate changes only apply to NEW transactions, not retroactively. Already-issued LYMX stays at the original rate.

Can't log in to dashboard — 2FA broken

Dashboard
Symptom: 2FA code works but immediately says "invalid."
Fix steps
  1. Check your phone time — 2FA is time-sensitive, off-by-30s clocks fail
  2. Try voice call instead: tap "Call instead" on 2FA screen, we read code aloud
  3. If 2FA app (Google Authenticator) is broken: contact biz-support@lymx.com with photo ID. We disable 2FA temporarily so you can log in and re-enroll.

Numbers in dashboard don't match my POS reports

Dashboard
Symptom: POS says $4,200 in LYMX-tagged sales, dashboard shows $4,150.
Likely causes
  1. Refunds — LYMX automatically deducts refunded transactions; POS may show pre-refund total
  2. Tipping — LYMX issues on subtotal (not tip); POS may show including tip
  3. Time zone mismatch — POS local time vs UTC settlement time
  4. Run reconciliation report: Dashboard → Reports → Reconcile vs POS. Shows the exact differences row-by-row.

Staff member can see things they shouldn't

Dashboard
Symptom: Cashier role can view financial reports.
Fix steps
  1. Dashboard → Staff → tap user → Role
  2. Set to "Cashier" (limited) or "Manager" (operational, no financials) or "Owner" (full)
  3. See role definitions for what each can do

Can't add a new location

Dashboard
Symptom: "Add location" button is grayed out.
Fix steps
  1. Multi-location requires verified owner role (not manager)
  2. Each new location is $50/mo on top of base $199
  3. If owner role is verified and still grayed: ensure your billing card on file is current. Stale card blocks add-location.

Settlement payout late this week

Settlement
Symptom: Wednesday payout normally hits Thursday morning, today is Friday and nothing.
Fix steps
  1. Check status.lymx.com — banking partner outage will be flagged there
  2. Check your bank info: Dashboard → Settings → Banking → confirm routing/account
  3. Hit holiday? Federal banking holidays push settlement back 1 business day
  4. Still missing 24h+ past expected: contact your success coach. We can manually expedite.

Settlement amount looks wrong

Settlement
Symptom: You expected $X, you got $Y.
Why settlements vary
  1. 3% LYMX network fee on issuance
  2. 80% redemption cost when LYMX issued elsewhere is redeemed at your Business
  3. Refunds and disputes
  4. Run Settlement Reconciliation: Dashboard → Reports → Settlement detail. Shows breakdown line-by-line.
  5. If still confused: file settlement dispute — we walk through the math with you.

Bank rejected the deposit

Settlement
Symptom: Settlement bounced back to LYMX.
Fix steps
  1. Almost always a wrong account/routing number
  2. Dashboard → Settings → Banking → re-enter and verify with micro-deposits
  3. Once verified, we resend the bounced settlement within 1 business day

Customer says they didn't earn LYMX from their visit

Customer
Symptom: Customer in your store complaining no LYMX appeared.
Fix in 60 seconds
  1. Check POS — if transaction was tagged with their phone/QR, LYMX was issued
  2. Have them pull-to-refresh their wallet
  3. If POS didn't capture their wallet: issue manual credit. Dashboard → Customer claims → enter phone + amount + LYMX. Hits their wallet in 60 seconds.
  4. This protects both sides — customer earns, you keep the relationship.

Customer wants to redeem LYMX but POS rejects

Customer
Symptom: "Insufficient balance" — but customer's wallet shows enough.
Likely causes
  1. Customer has "pending" LYMX (issued <24 hours ago, not yet settled). Show them wallet → Available vs Pending breakdown.
  2. Daily redemption cap — your Business can set per-day per-customer caps. Check Dashboard → Settings → Redemption rules.
  3. If neither: it's a sync issue. Try re-scanning QR. If it persists, accept payment normally — issue retroactive LYMX.

Customer disputes a charge — what do I do?

Customer
Symptom: A LYMX dispute appears in your dashboard.
Fix steps
  1. Dashboard → Disputes → tap to view
  2. Pull POS receipt for the disputed transaction (search by date + amount)
  3. Upload receipt photo + your written response
  4. LYMX support reviews and resolves within 7 days
  5. See full dispute handling guide

POS keeps prompting for "reconnect to LYMX"

POS
Symptom: Every 5 minutes, POS shows reconnect prompt.
Fix steps
  1. Network is unstable — check POS internet
  2. POS firmware out of date — update to latest version (check POS vendor)
  3. LYMX integration token may be old: Dashboard → Integrations → Refresh token

Promo I scheduled never went live

Dashboard
Symptom: "2x LYMX weekend" was scheduled, customers say it didn't apply.
Fix steps
  1. Dashboard → Promos → check status. "Scheduled" hasn't started, "Live" has.
  2. Confirm promo time zone — set to America/Los_Angeles by default. Off-hours runs in early Pacific time.
  3. Confirm promo wasn't capped: if you set "first 100 customers," it stops after 100.
  4. If promo was active but POS didn't apply: check POS-LYMX sync was working that hour.

I want to leave the network — how does cancellation work?

Dashboard
Symptom: Looking to discontinue LYMX.
How cancellation works
  1. Dashboard → Settings → Cancel subscription. Initiates 90-day wind-down.
  2. Day 1: stop issuing new LYMX. Day 90: stop accepting LYMX redemption.
  3. During wind-down, your listing shows "Leaving the LYMX network — still accepting LYMX."
  4. Outstanding LYMX you issued is auto-capped to your settlement reserve. You'll receive itemized exit statement.
  5. See refund policy section 5b for full details.

Selling my Business — does LYMX transfer to new owner?

Dashboard
Symptom: You're selling and want to know what happens to LYMX.
Fix steps
  1. Yes — LYMX is a transferable asset if the new owner stays on the network
  2. Dashboard → Settings → Ownership transfer. Submit new owner's info. We send them activation link.
  3. Buyer continues at the same rate, same customer base, same LYMX outstanding
  4. If new owner exits the network instead: 90-day wind-down activates from sale date
  5. See business agreement section 10b for full details.
