LYMX API

Issue and accept LYMX programmatically. Sync transactions from your POS. Build webhooks against wallet events. The same engine that powers our Web POS, exposed as a JSON REST API.

Beta status: available today to Founding 25 Businesses with custom POS integrations. Public release summer 2026 alongside our Square / Toast / Clover partner integrations. Endpoints documented here are stable; we'll version any breaking change with a 90-day deprecation window.

Request access

Email api@getlymx.com with your Business name, Business ID (from your dashboard), and a 1-line description of what you're building. We respond within 1 business day during beta.

Once approved, you get:

  • 1 sandbox key — for development and testing. Sandbox transactions don't settle.
  • 1 live key — for production. All actions affect real wallet balances and Business settlement.
  • 1 webhook signing secret — to verify webhook payloads we send to you.

Authentication

Every request includes your API key in the Authorization header. Keys are 32-char tokens prefixed with lymx_live_ or lymx_test_.

cURL
# Production curl https://api.getlymx.com/v1/balance \ -H "Authorization: Bearer lymx_live_xxxxxxxxxxxxxxxx" # Sandbox curl https://api.getlymx.com/v1/balance \ -H "Authorization: Bearer lymx_test_xxxxxxxxxxxxxxxx"
Never commit API keys to source. Use environment variables or a secrets manager. We auto-rotate any key we find publicly leaked.

Your first call

Make sure your key works by calling the balance endpoint for a test customer:

cURL
curl https://api.getlymx.com/v1/customers/cus_demo123/balance \ -H "Authorization: Bearer lymx_test_..." \ -H "Accept: application/json"
Node.js
import Lymx from '@lymx/node'; const lymx = new Lymx(process.env.LYMX_KEY); const balance = await lymx.customers.balance('cus_demo123'); console.log(balance); // => { customer_id: 'cus_demo123', lymx: 2450, usd_value: 24.5, ... }
Python
import lymx lymx.api_key = os.environ["LYMX_KEY"] balance = lymx.Customer.balance("cus_demo123") print(balance.lymx, balance.usd_value) # => 2450 24.5

Issue LYMX — give a customer LYMX after a sale

POST /v1/transactions/issue Issue LYMX to a customer's wallet after they pay for a purchase.

Trigger after the customer completes a sale at your POS. Auto-applies any active promotion multipliers configured in your Promotions dashboard.

Request body

FieldTypeDescription
customer_phoneRequiredstringE.164-formatted phone number of the customer (e.g. +17025550181).
subtotal_centsRequiredintegerPre-tax sale subtotal in cents. We multiply by your locked rate to compute LYMX earned.
idempotency_keyRequiredstringUnique key per transaction. Repeat calls with the same key return the original result without re-issuing.
tip_centsOptionalintegerTip amount in cents. Tips don't earn LYMX by default.
itemsOptionalarrayLine items, used by item-targeted promotions. See Items reference.

Example

cURL
curl -X POST https://api.getlymx.com/v1/transactions/issue \ -H "Authorization: Bearer lymx_live_..." \ -H "Content-Type: application/json" \ -d '{ "customer_phone": "+17025550181", "subtotal_cents": 1450, "idempotency_key": "order_2026-04-29_004086" }'

Response (201)

JSON
{ "id": "tx_2zR8aQpL", "type": "issue", "customer_id": "cus_8jN4b2", "lymx_issued": 145, "base_rate": 5, "multiplier": 2.0, "promotion_id": "promo_happyhour_w17", "created_at": "2026-04-29T16:08:42Z" }

Accept LYMX — deduct LYMX from a customer's wallet as payment

POST /v1/transactions/redeem Apply LYMX from a customer's wallet to a sale.

Two-step: first call /v1/transactions/redeem/quote to get a quote and a confirmation URL the customer scans. After they confirm, the redemption settles automatically and you receive a webhook.

For low-value redemptions under the customer's auto-approve threshold (default $20), the quote auto-confirms and you can skip the second step.

GET /v1/customers/{phone_or_id}/balance Returns the customer's current LYMX balance.

List transactions

GET /v1/transactions List all transactions for your Business, paginated.

Webhooks

We send webhooks to your endpoint when state changes. Configure your endpoint URL and signing secret in the Business dashboard.

Events

  • transaction.created — a new issue or redemption posted
  • transaction.disputed — a customer or Business filed a dispute
  • customer.balance.updated — a customer's balance changed
  • settlement.statement.ready — your monthly settlement statement is ready
  • promotion.activated / promotion.deactivated — promo multiplier changed state

Verify a webhook

Each webhook includes an X-LYMX-Signature header — an HMAC-SHA256 of the raw body using your signing secret. Verify before processing:

Node.js
import crypto from 'crypto'; function verify(rawBody, signature, secret) { const expected = crypto .createHmac('sha256', secret) .update(rawBody) .digest('hex'); return crypto.timingSafeEqual( Buffer.from(signature), Buffer.from(expected) ); }

Error codes

StatusCodeMeaning
400invalid_requestMissing or malformed parameters.
401authentication_failedInvalid or missing API key.
402insufficient_balanceCustomer doesn't have enough LYMX for the redemption.
404not_foundThe referenced resource doesn't exist.
409idempotency_conflictSame idempotency_key with different parameters.
429rate_limit_exceededYou've hit the rate limit. Retry after the time in the Retry-After header.
500internal_errorSomething broke on our end. Check status.getlymx.com.

Rate limits

Default beta limits per Business:

  • Read endpoints (GET balance, list transactions): 100 req/min
  • Write endpoints (issue, redeem): 30 req/min sustained, 120 req/min burst

Need higher limits for a launch event or large sale? Email api@getlymx.com 48h ahead.
