5 staff roles, choose right

Your LYMX dashboard has 5 distinct roles, each with different visibility and powers. Get this right at setup — you don't want a part-time cashier accidentally able to issue refunds or change your subscription.

👑

Owner

Level 5 · Full

You. Full control. Billing, settings, all reports.

👨‍💼

Manager

Level 4 · Operational

Your store manager. Can run promos, handle disputes, hire staff.

🧾

Cashier

Level 3 · POS

Front-of-house. Scan QR, accept LYMX, see today's tip jar.

👀

Viewer

Level 2 · Read-only

Accountant or partner. See reports, can't change anything.

🤖

API

Level 1 · Programmatic

Automated integrations. Token-only, scoped permissions.

Real-world scenarios

I'm a single-location restaurant with a manager and 4 cashiers

Owner role: you. Manager role: your manager. Cashier role: each cashier (own login).

Why this mattersIf a cashier shares their login with another, you lose audit trail. Each cashier with their own login = you can see who scanned what.

I have an external accountant who needs reports

Add them as Viewer role.

Why Viewer not ManagerAccountant doesn't need to run promos or hire staff. Viewer is read-only — safest level for non-employees.

My business partner co-owns the Business

Two options: dual Owner roles, or one Owner + one Manager.

RecommendationTwo Owner roles if both partners pay/share equally. Owner role can change billing — you both should be able to. Mutual trust required.

I'm temporarily delegating to a manager while I'm out of country

Don't promote to Owner. Use Manager role + temporary expanded permissions.

Best practiceSettings → Staff → Manager → "Expand permissions: 14 days." Auto-revokes back to standard Manager when you return. Audit trail of every elevated action.

I want to integrate with my own loyalty system via API

Generate API role with scoped permissions.

HowSettings → API → Create token. Pick scopes (read transactions only / read+write / etc.). Tokens are revocable anytime.

A cashier left and I forgot to remove them

You can — and should — revoke immediately.

HowSettings → Staff → tap user → Remove access. Or use shortcut: Settings → "Removed (last 30 days)" to audit your offboarding hygiene.

⚠️ Danger zone — only Owner can do these

Change LYMX rate · Change billing card · Cancel subscription · Add new location · Initiate ownership transfer · Disable 2FA. If your manager pressures you to give them Owner access for any of these — say no. They can ask, but only you should approve.

Setup walkthrough

  1. Dashboard → Settings → Staff → Add new
  2. Enter their email or phone — invite goes via SMS
  3. Pick role (Owner / Manager / Cashier / Viewer / API)
  4. Optional: limit by location (multi-location only) or by hours
  5. They click invite link, set their own PIN, they're in
  6. You receive confirmation when they activate
