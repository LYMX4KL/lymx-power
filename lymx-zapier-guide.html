Set up your first Zap in 10 minutes

  1. Get your LYMX API key. Dashboard → Settings → API → Create new key. Scope to "events read-only" for safety.
  2. Sign up for Zapier at zapier.com. Free tier covers 5 Zaps. Paid tiers ($20+/mo) for more.
  3. Search "LYMX" in Zapier's app directory. (Currently in beta — request access via partnerships@lymx.com if not visible to you yet.)
  4. Click "Make a Zap" → choose LYMX as trigger. Pick the event from the 8 listed above.
  5. Connect your LYMX account. Paste your API key. Test connection.
  6. Choose action app. Slack, Sheets, Mailchimp, etc. Connect that account.
  7. Map fields. Drag LYMX trigger fields into the action's expected fields.
  8. Test the Zap. Make a test transaction at your Business. Confirm action fires.
  9. Turn it on. You're live. Zap runs forever (or until you turn it off).

FAQ

Is the LYMX Zapier app currently public?

Beta as of August 2026. Public release scheduled for Q1 2027. Beta access available to all LYMX Businesses by request: partnerships@lymx.com.

How real-time are the triggers?

Sub-30-second latency for transaction.completed and redemption.completed. Up to 5-minute latency for derived events like customer.lapsed (which run on a schedule).

Does my Zap count against any LYMX rate limit?

Reasonable use is fine. We allow up to 1,000 events per day per Business at no extra charge. Heavy users (10K+ events/day) may need an enterprise tier.

Can I trigger LYMX actions from Zapier (issue LYMX, send notifications)?

Yes — beyond reading events, the LYMX Zapier app supports actions: issue retroactive LYMX, send wallet notification, schedule promo. Each action requires elevated API permissions.

What about HIPAA / data privacy concerns?

Zaps run on Zapier's servers. Customer data flowing through them is subject to Zapier's privacy policy. If you're handling sensitive customer data, prefer direct webhooks (data stays in your control).

Can my Local Manager set up Zaps?

API key creation is Owner-only. Local Manager can use a Zap once configured, but can't create new ones (without elevated permission).
