Authentication

Pass your API key in the Authorization header. Sandbox keys start with lymx_test_; live keys with lymx_live_.

cURL
curl https://api.getlymx.com/v1/balance \ -H "Authorization: Bearer lymx_test_xxxxxxxxxxxxxxxx" \ -H "Accept: application/json"
Node.js
import Lymx from '@lymx/node'; const lymx = new Lymx('lymx_test_xxxxxxxxxxxxxxxx'); // Sandbox vs live is auto-detected from the key prefix
Python
import lymx lymx.api_key = 'lymx_test_xxxxxxxxxxxxxxxx'
Ruby
require 'lymx' Lymx.api_key = 'lymx_test_xxxxxxxxxxxxxxxx'
PHP
<?php \Lymx\Lymx::setApiKey('lymx_test_xxxxxxxxxxxxxxxx');
Never commit API keys. Use environment variables. We auto-rotate any key found in a public repo.

POST/v1/transactions/issue

Issue LYMX to a customer's wallet after a sale. Idempotent via idempotency_key header.

cURL
curl -X POST https://api.getlymx.com/v1/transactions/issue \ -H "Authorization: Bearer lymx_test_..." \ -H "Content-Type: application/json" \ -H "Idempotency-Key: order_2026-04-29_004086" \ -d '{ "customer_phone": "+17025550181", "subtotal_cents": 1450, "items": [ {"name": "Latte", "price_cents": 575}, {"name": "Scone", "price_cents": 875} ] }'
Node.js
const tx = await lymx.transactions.issue({ customer_phone: '+17025550181', subtotal_cents: 1450, items: [ { name: 'Latte', price_cents: 575 }, { name: 'Scone', price_cents: 875 } ] }, { idempotencyKey: 'order_2026-04-29_004086' }); console.log(tx.lymx_issued); // => 145 (with 2x Happy Hour multiplier)
Python
tx = lymx.Transaction.issue( customer_phone="+17025550181", subtotal_cents=1450, items=[ {"name": "Latte", "price_cents": 575}, {"name": "Scone", "price_cents": 875}, ], idempotency_key="order_2026-04-29_004086", ) print(tx.lymx_issued) # => 145
Ruby
tx = Lymx::Transaction.issue( customer_phone: '+17025550181', subtotal_cents: 1450, items: [ { name: 'Latte', price_cents: 575 }, { name: 'Scone', price_cents: 875 } ], idempotency_key: 'order_2026-04-29_004086' ) puts tx.lymx_issued # => 145
PHP
$tx = \Lymx\Transaction::issue([ 'customer_phone' => '+17025550181', 'subtotal_cents' => 1450, 'items' => [ ['name' => 'Latte', 'price_cents' => 575], ['name' => 'Scone', 'price_cents' => 875] ], 'idempotency_key' => 'order_2026-04-29_004086' ]); echo $tx->lymx_issued; // 145

POST/v1/transactions/redeem

Two-step pattern. Step 1: request a quote → returns a redemption URL the customer scans. Step 2: customer confirms → webhook fires → you receive the settled amount.

Step 1: Request quote
curl -X POST https://api.getlymx.com\ /v1/transactions/redeem/quote \ -H "Authorization: Bearer lymx_test_..." \ -d '{ "customer_phone": "+17025550181", "amount_cents": 2400 }'
Returns
{ "quote_id": "qt_8sLp", "customer_qr_url": "https://...", "expires_at": "2026-04-29...", "auto_confirm": false }

GET/v1/customers/{phone_or_id}/balance

Returns the customer's current LYMX balance. Pass either an E.164 phone or a cus_* ID.

curl https://api.getlymx.com/v1/customers/+17025550181/balance \ -H "Authorization: Bearer lymx_test_..."
Response
{ "customer_id": "cus_8jN4b2", "phone": "+17025550181", "lymx": 2450, "usd_value": 24.50, "updated_at": "2026-04-29T16:14:01Z" }

GET/v1/transactions

List your Business's transactions. Cursor-paginated, default page size 50.

curl "https://api.getlymx.com/v1/transactions?limit=10&type=issue&starting_after=tx_2zR8aQpL" \ -H "Authorization: Bearer lymx_test_..."

Webhook signature verification

Every webhook includes X-LYMX-Signature — an HMAC-SHA256 of the raw body using your signing secret.

Node.js · Express
import express from 'express'; import crypto from 'crypto'; const app = express(); const SECRET = process.env.LYMX_WEBHOOK_SECRET; app.post('/webhook', express.raw({ type: '*/*' }), (req, res) => { const sig = req.headers['x-lymx-signature']; const expected = crypto .createHmac('sha256', SECRET) .update(req.body) .digest('hex'); if (!crypto.timingSafeEqual(Buffer.from(sig), Buffer.from(expected))) { return res.status(401).send('invalid signature'); } const event = JSON.parse(req.body); // handle event.type ... res.send('ok'); });
Python · Flask
import hmac, hashlib, os, json from flask import Flask, request, abort app = Flask(__name__) SECRET = os.environ["LYMX_WEBHOOK_SECRET"].encode() @app.post("/webhook") def webhook(): sig = request.headers["X-LYMX-Signature"] expected = hmac.new(SECRET, request.data, hashlib.sha256).hexdigest() if not hmac.compare_digest(sig, expected): abort(401) event = json.loads(request.data) # handle event["type"] ... return "ok"
Ruby · Sinatra
require 'sinatra' require 'openssl' SECRET = ENV['LYMX_WEBHOOK_SECRET'] post '/webhook' do body = request.body.read sig = request.env['HTTP_X_LYMX_SIGNATURE'] expected = OpenSSL::HMAC.hexdigest('sha256', SECRET, body) halt 401 unless Rack::Utils.secure_compare(sig, expected) event = JSON.parse(body) # handle event['type'] ... 'ok' end
PHP
<?php $secret = $_ENV['LYMX_WEBHOOK_SECRET']; $body = file_get_contents('php://input'); $sig = $_SERVER['HTTP_X_LYMX_SIGNATURE']; $expected = hash_hmac('sha256', $body, $secret); if (!hash_equals($expected, $sig)) { http_response_code(401); exit('invalid signature'); } $event = json_decode($body, true); // handle $event['type'] ... echo 'ok';

Error handling

Wrap calls in try/catch and check status codes. Common errors: 401 (auth), 402 (insufficient balance), 409 (idempotency conflict), 429 (rate limit).

Node.js
try { const tx = await lymx.transactions.issue({...}); } catch (err) { if (err.code === 'insufficient_balance') { // Customer doesn't have enough LYMX } else if (err.code === 'rate_limit_exceeded') { const retryAfter = err.headers['retry-after']; // Wait retryAfter seconds + retry } else { throw err; } }
