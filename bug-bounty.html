Scope

What's in vs. out of scope for the bug bounty program.

✓ In scope

  • getlymx.com and all subdomains
  • LYMX wallet web app
  • LYMX Business dashboard + POS
  • LYMX Partner dashboard + downstream
  • Public API (api.getlymx.com)
  • Settlement & commission backend
  • Authentication / authorization flows
  • Stripe webhook handlers (LYMX side)

✗ Out of scope

  • Third-party services (Stripe, Square, Toast — report directly to them)
  • Social engineering of LYMX staff
  • Physical attacks on LYMX offices
  • Denial-of-service attacks
  • Spam or rate-limit findings <1000 req/min
  • Vulnerabilities in beta features (announced)
  • Self-XSS without other impact
  • Issues only affecting outdated browsers

Rules & responsible disclosure

  • No data exfiltration. Find the vulnerability, document it, but don't extract customer data. Take a screenshot showing the issue, not the data itself.
  • No service disruption. Don't run scanners that could affect uptime. If unsure, ask first.
  • Don't pivot. If you find an entry point, document it but don't keep going to find more bugs through that entry point. We'll fix it; you get credit for the original finding.
  • One report per issue. If you find multiple related bugs, file them separately.
  • 90-day disclosure window. We commit to fixing critical bugs within 30 days, others within 90 days. After fix is deployed, you can publish the finding (with our acknowledgment).
  • Don't sell findings. If you find a bug here and try to sell it elsewhere first, you forfeit the bounty.

Frequently asked questions

What if I find a bug before signing up for the program?

You don't need to "sign up." The program is open. Submit a finding, follow the rules, and you'll get paid if it's valid.

Can I be anonymous?

Yes. Tell us when you submit — we'll use a pseudonym in the hall of fame and pay via your preferred method (PayPal, BTC, gift card, or charity donation).

What payment methods do you support?

USD via PayPal, ACH (US), Wise (international), or BTC. We can also donate the bounty to a charity of your choice. Tax forms: we issue 1099-MISC for US researchers earning $600+ annually.

What if multiple researchers find the same bug?

First valid report gets full bounty. Subsequent reports of the same bug are acknowledged but not paid. Timestamp on email is the tiebreaker.

Can I publicly disclose a bug after you fix it?

Yes — please coordinate timing with us. Standard practice: 30 days after fix is deployed, or 90 days from initial report (whichever is later). We'll happily acknowledge and link to your write-up.

I found a bug in a tool you depend on (Stripe, Postgres, etc.). Should I report it to you?

Report it directly to the upstream maintainer. We'll happily review your finding and amplify if you'd like — but the bounty would come from them, not us. We pay only for findings in LYMX-controlled code.
